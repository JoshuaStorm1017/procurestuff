# ProcureStuff

ProcureStuff is a free, self-hosted procurement intake and purchase order workspace. It replaces paid spend control planes with local persistence, full CRUD, workflow actions, audit history, analytics, import, and open exports.

## Features

- Purchase request intake with requester, department, category, supplier, budget, amount, need date, justification, status, and tags.
- Supplier directory, approval tasks, purchase orders, and budget tracker.
- Search, filtering, sorting, detail inspectors, modal editors, archive, and restore across every collection.
- Dashboard metrics for active requests, requested spend, pending approvals, and supplier renewals due soon.
- Workflow actions for advancing requests, approving tasks, and marking purchase orders received.
- Workspace settings for procurement policy, approval cadence, spend categories, and finance reviewers.
- JSON and CSV exports for every collection, full workspace export, and workspace import.
- Local audit activity generated by create, update, archive, restore, import, and workflow actions.

## Stack

- Node 24 HTTP server.
- Static HTML, CSS, and JavaScript app.
- Atomic JSON persistence in `data/procurestuff.json`.
- Node built-in test runner.
- No paid procurement platform credentials required for core usage.

## Setup

```bash
npm install
npm run dev
```

Then open `http://127.0.0.1:4162`.

To store data elsewhere:

```bash
PROCURESTUFF_DATA=/absolute/path/procurestuff.json npm run dev
```

## Architecture

- `server/config.js` defines product metadata, fields, workflows, metrics, and seed data.
- `server/domain.js` validates and mutates workspace state.
- `server/store.js` reads and writes the local JSON store atomically.
- `server/routes.js` exposes state, CRUD, workflow, import, and export APIs.
- `public/app.js` renders the dashboard, workbenches, forms, settings, and import/export controls.
- `tests/` covers domain behavior and API routes without opening a network socket.

## Deployment Notes

ProcureStuff can run anywhere Node 24 can run. Persist the `data/` directory or set `PROCURESTUFF_DATA` to a durable mounted path. Reverse proxying behind HTTPS is recommended for shared team access.
